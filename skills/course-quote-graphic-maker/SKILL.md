# 課程圖文製作

## 必讀規格

在產出任何圖文前，完整閱讀 [references/sop.md](references/sop.md)。以其中的內容忠實性、版型、日期、人物安全區與驗收規則為準。

## 目標

把每一節課的現場照片與逐字稿，直接製作成一張 16:9、2048 × 1152、高解析 PNG。保留現場紀錄真實性，提煉一個核心觀點，並維持同系列一致的沉靜、精緻、文化感。

## 執行流程

1. 接收照片、逐字稿、課程實際日期與品牌名稱。若沒有品牌名稱，使用 `課程精選`。
2. 若使用者沒有明確提供日期，先從本輪訊息、前後文、課程表、檔名與照片 metadata 判定；只有無法確認或互相矛盾時才詢問。
3. 完整消化逐字稿，修正明顯辨識錯誤與口語贅詞，但不增加講者沒有表達的觀點。
4. 建立核定文字稿：一個核心主句，加上一段簡短承接或一至兩個思辨問題。
5. 檢視原始照片，判斷人物位置、清晰度、表情、手勢與可用文字空間。
6. 保留原始照片，從工作副本開始。用裁切、縮放、定位、色調與 HTML/CSS 排版完成圖文。
7. 依下方配置建立 JSON，使用 [scripts/build_course_graphic.mjs](scripts/build_course_graphic.mjs) 與 [assets/course-graphic-template.html](assets/course-graphic-template.html) 渲染 PNG。
8. 實際開啟成品，以縮圖與原尺寸各檢查一次；未通過就重排後再輸出。
9. 在對話中顯示實際預覽、下載連結、核心主句與課程日期。

## 核定文字稿

固定內容欄位：

- `brand`：使用者的品牌或系列名稱，限單行 18 個字元；未提供時使用 `課程精選`。
- `headline`：約 12–30 個中文字，原則上不超過三行。
- `reflection`：約 30–70 個中文字，原則上三至五行；問題行標為 `question`。

不要在圖面顯示第幾天、第幾堂、時段、人物角色或分享者姓名；即使使用者提供，也只作背景理解，不放入頁首或其他位置。

精簡文字時：

- 保留講者原意與語氣。
- 使用繁體中文與正確全形標點。
- 不為了文學感改寫成講者沒有說過的主張。
- 不把整段逐字稿塞進畫面。
- 不再產生左下角收尾金句。

## 日期規則

日期只顯示 `YYYY.MM.DD`，並直接取代左下角短句的位置。

來源優先順序：

1. 使用者明確提供的課程日期。
2. 已知課程表或活動資訊。
3. 照片原始拍攝日期。
4. 仍無法確認時詢問使用者。

不得加「西元」、星期、前綴或任何補充短句；不得自行以製作日代替課程日。

## 照片與人物規則

- 預設左文右人，講者位於右側明亮區。
- 只用整張原照的裁切、縮放與定位調整人物位置，不切出人物後生成搬移。
- 暗色蒙板必須在碰到人物前完全淡出；臉、頭髮、上半身、手與麥克風不得被壓暗。
- 不覆寫原始照片。
- 不以生成式版本取代正式紀錄照。若生成探索改變五官、年齡、姿勢、服裝、名牌、投影或牆面文字，立即淘汰。
- 中文一律另行排版，不讓影像生成工具直接生成中文字。
- 若照片無法同時容納文字與人物，請使用者提供替代照片，不勉強遮人或壓字。

## 渲染配置

先建立 UTF-8 JSON，例如：

```json
{
  "brand": "你的品牌",
  "headline": "所有的一切，\n都從「看見」開始。",
  "highlight": "「看見」",
  "reflection": [
    {"text": "我們每天都在鏡子裡看見自己，", "kind": "body"},
    {"text": "卻未必真正看見——", "kind": "body"},
    {"text": "我是怎麼過著自己？", "kind": "question"},
    {"text": "正處在什麼樣的生命狀態？", "kind": "question"}
  ],
  "date": "2026.07.17",
  "photo": {
    "scale": 1.12,
    "positionX": "0%",
    "positionY": "50%",
    "transformOrigin": "left center"
  }
}
```

先呼叫工作區依賴定位工具取得 Node、Playwright 與可用瀏覽器路徑，再執行：

```bash
<node> scripts/build_course_graphic.mjs \
  --config <config.json> \
  --photo <原始照片> \
  --output <輸出.png> \
  --playwright-module <playwright/index.mjs> \
  --chrome <Chrome執行檔>
```

`--playwright-module` 與 `--chrome` 在環境可自行找到時可省略。若需修訂，建立 `v02`、`v03`，不要覆蓋已確認版本。

## 驗收

只有以下全部通過才交付：

- 左上角只有使用者指定的品牌或系列名稱，沒有堂次、時段、人物角色或姓名。
- 主句與日期逐字正確。
- 左下角只有 `YYYY.MM.DD`。
- 沒有「西元」與額外短句。
- 人物右置、明亮，沒有蒙板薄膜。
- 文字無裁切、重疊、亂碼或簡繁混用。
- 人物、投影與現場內容未被生成式工具改變。
- PNG 為 2048 × 1152，已實際開啟檢查。

## 修正與定稿

收到校正時，先指出真正失準點，只修改使用者指定的部分，重新渲染並顯示完整預覽。版本使用 `v01`、`v02`；使用者確認後另存 `_定稿_vNN`。
